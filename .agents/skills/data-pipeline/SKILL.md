### The Full Data Workflow (Pipeline)

This workflow ensures that **Source Provenance** is never lost. Every data point retains a link to the original URL and text snippet.

### **Phase 1: Ingestion (The Railway Worker)**
*Trigger: Runs every hour via Railway Cron.*

1.  **Discovery (GDELT):** Fetch new URLs from `digi24.ro`, `antena3.ro`, `maszol.ro`, etc.
2.  **Scraping & Fingerprinting:**
    *   Use `trafilatura` to get the text.
    *   **The Source Lock:** Immediately assign a `source_id` (UUID). Store the `original_url`, `author`, `publication_date`, and `raw_html_hash`.
    *   *Constraint:* If scraping fails, the item is tagged `status: "failed_scrape"` but kept in DB for audit.
3.  **Vectorization:** Generate 1536D vectors using OpenAI `text-embedding-3-small`.
    *   *Provenance:* The vector is stored with `source_id` attached.

### **Phase 2: AI Structuring (The Gemini Queue)**
*Trigger: Runs immediately after scraping, strictly throttled.*

1.  **The Rate-Limit Queue:** The Python worker pushes articles into an internal queue.
    *   It processes them **one by one** (or in small batches of 3) to stay under the **Gemini Free Tier (RPM)** limits.
2.  **Structuring (LangExtract + Gemini Flash Lite):**
    *   Extracts Entities, Claims, and Stance.
    *   **The Grounding:** LangExtract returns `start_char` and `end_char` for every claim.
    *   *Provenance:* The structured JSON includes the `source_id` and the exact text snippet.
3.  **Push to Convex:**
    *   Insert `Article` (with Source Metadata).
    *   Insert `Claims` (linked to `Article` via ID).
    *   Insert `Vectors`.

### **Phase 3: Spatial Math (Batch Processing)**
*Trigger: Runs every 6 hours on Railway.*

1.  **Clustering:** Pull vectors -> Run **HDBSCAN**.
    *   *Provenance:* The resulting `Cluster_ID` is written back to the `Article` table.
2.  **Projection:** Run **PCA** (Time Axis) + **UMAP** (Semantic Axis).
    *   *Output:* `[x, y, z]` coordinates stored in Convex.

### **Phase 4: Impact & Profiling (Weekly/Nightly)**
*Trigger: Weekly (Impact) / Nightly (Profiles).*

1.  **Social Sweep:** Fetch Facebook/X shares via Apify.
    *   *Provenance:* Update the `impact_score` on the specific `source_id`.
2.  **Profile Aggregation:** Convex scans the `Claims` table to build Newspaper Profiles (Bias/Ghosting).

## 3. The 7 Core Features (UI/UX)

### **1. The "Morning Briefing" (Entry Point)**
*   **UX:** A clean, text-based daily summary generated by Gemini Lite: *"Good morning. Today's narrative divergence is highest on the 'Tax Code' cluster."*
*   **Controls:** Global Time Window Toggle: **[ 24h ] [ 48h ] [ 96h ]**.

### **2. The 3D Event Universe (Semantic River)**
*   **Visual:** A **React Three Fiber** galaxy.
    *   **X-Axis:** Locked to Time (Past -> Present).
    *   **Y/Z Axes:** Semantic similarity (Topics).
    *   **Node Size:** Weighted by Social Media Impact (Virality).
*   **Interaction:**
    *   **Hover:** "True Neighbor Tethers" (lines connecting mathematical neighbors).
    *   **Click:** "Cluster Explode" (Zoom in, re-project cluster for clarity).

### **3. The Article X-Ray (Relational Graph)**
*   **Visual:** **React Flow** network graph above the text.
*   **Source Lock:** Nodes = Entities. Edges = Claims.
    *   **Interaction:** Hovering an edge highlights the *exact sentence* in the article text below (using the stored character indices).

### **4. Consensus vs. Spin Timeline**
*   **Visual:** **D3.js** Diff Timeline.
*   **Logic:** Center line = Baseline Facts (Consensus). Branches = Editorial Spin (Newspapers).
    *   *Green Highlight:* Added Context.
    *   *Red Strikethrough:* Omitted Facts.

### **5. The Typology Divide (National vs. Local)**
*   **Visual:** Split-pane dashboard.
*   **Logic:** Compare the Top Entities/Stances of Bucharest outlets vs. Local/Minority outlets.
*   **Metric:** "Divergence Score" (0-100%).

### **6. Systematic Bias & Ghosting Tracker**
*   **Visual:** **D3.js** Heatmap.
*   **Logic:**
    *   **Ghosting:** Did Outlet X ignore a cluster that fits their historical topic profile? (Propensity-weighted).
    *   **Bias:** Net Hostility Score towards specific politicians.

### **7. The Multi-Outlet Comparison Engine**
*   **Visual:** **TanStack Table** (Master Matrix).
*   **UX:** Select up to 5 outlets.
    *   **Visual Mode:** Select max 3 widgets (e.g., Speed Dial + Bias Radar).
    *   **Table Mode:** Full data grid with conditional formatting (Red = High Sensationalism).
